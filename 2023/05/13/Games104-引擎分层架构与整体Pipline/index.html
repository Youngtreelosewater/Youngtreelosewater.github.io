引擎分层架构与整体Pipline

架构分层

  • 工具层 Tool
    • 编辑器
  • 功能层 Function
    • 可见的、可动的、可玩的
  • 资源层 Resource
  • 核心层 Core
  • 平台层 Platform

![[Pasted image 20230412142648.png]]

为什么分层？

  • ![[Pasted image 20230412153628.png]]
  • 解耦Decpling 降低复杂度Complexity
    • 下层 独立于上层
    • 上层 不知道 下层 实现implemented
  • 接受需求时，只需动能层，底层稳定
  • tick：只允许上层调用下层

资源层

resource 到 Access

  • 给资源发放guid，资产的身份识别号
  • 实时资产管理器
    • handle system
  • 生命周期管理
    • GC 垃圾回收
    • 延时加载

功能层

内容最多
Ticks 周期

  • 一个Tick 处理所有该做的事情，运转所有的齿轮
  • tickMain 函数（可能不叫这名）
    • Tick Logic 逻辑帧
      • 物理引擎
    • Tick Render 渲染帧
      • 渲染引擎
  • 多线程 一核有难，七核围观
    • 并行，先后顺序是难点
    • 多核未来趋势 核心代码考虑多核

核心层

数学库

  • 物理所用数学库非常高深，一般的简单
    效率 Efficiency
  • 倒数平方根 耗，用大致解
  • SIMD 单指令流多数据流
    数据结构，为上层所有数据服务
  • ![[Pasted image 20230412150820.png]]
  • Array 存数据；二叉树 查找等等
  • 当超过存储空间，会倍数增长
  • 追求访问效率，没有内存碎片
    内存管理
  • ![[Pasted image 20230412151852.png]]
  • Cpu的缓存，相同主频，cashe大，吞吐数据更快，
  • PMR 局部性原理
    • C++等新功能的底层逻辑
    • 把数据放一起
    • 访问数据尽可能顺序访问
    • 读写的时候 一起去读写
  • CORE
    • ![[Pasted image 20230412152419.png]]
    • 写Core 一般资深

平台层

平台之间的图形库差异性

  • ![[Pasted image 20230412152702.png]]
  • 引擎水平分水岭

工具层

编辑器

  • 效率
    DCC Digital Content Creation
  • Asset Conditioning Pipeline 导出器与导入器
